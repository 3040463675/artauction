# 修复 AuctionDetail.vue 乱码问题

## 问题原因
文件中的中文被保存为问号，通常是文件编码不是 UTF-8 导致的。

## 解决方法

### 方法 1：在 Cursor/VSCode 中修复

1. **打开文件**
   - 在 Cursor 中打开 `frontend/src/views/AuctionDetail.vue`

2. **检查并更改编码**
   - 查看编辑器右下角的编码显示（可能显示为 `GBK`、`GB2312` 或其他）
   - 点击编码，选择 **"通过编码重新打开"**
   - 选择 **UTF-8**
   - 如果文件内容显示正常，说明编码已修复

3. **如果内容还是问号，手动替换**

### 方法 2：使用 Node.js 脚本修复

创建一个 `fix-chinese.js` 文件：

```javascript
const fs = require('fs');
const path = require('path');

const filePath = path.join(__dirname, 'frontend/src/views/AuctionDetail.vue');
let content = fs.readFileSync(filePath, 'utf8');

// 替换所有问号为正确中文
const replacements = {
  '<el-breadcrumb-item :to="{ path: \'/\' }">??</el-breadcrumb-item>': '<el-breadcrumb-item :to="{ path: \'/\' }">首页</el-breadcrumb-item>',
  '<el-breadcrumb-item :to="{ path: \'/explore\' }">??</el-breadcrumb-item>': '<el-breadcrumb-item :to="{ path: \'/explore\' }">探索</el-breadcrumb-item>',
  '<el-breadcrumb-item>????</el-breadcrumb-item>': '<el-breadcrumb-item>竞拍详情</el-breadcrumb-item>',
  // ... 添加更多替换
};

Object.keys(replacements).forEach(key => {
  content = content.replace(new RegExp(key.replace(/[.*+?^${}()|[\]\\]/g, '\\$&'), 'g'), replacements[key]);
});

fs.writeFileSync(filePath, content, 'utf8');
console.log('修复完成！');
```

然后运行：`node fix-chinese.js`

## 预防措施

1. **设置 Cursor/VSCode 默认编码为 UTF-8**
   - 打开设置（Ctrl+,）
   - 搜索 `files.encoding`
   - 设置为 `utf8`

2. **检查文件编码**
   - 编辑文件时，注意右下角编码显示
   - 如果显示非 UTF-8，立即切换

3. **使用 `.editorconfig`**
   在项目根目录创建 `.editorconfig`：
   ```
   root = true
   [*]
   charset = utf-8
   ```
